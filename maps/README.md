# BX1 map generators

Python scripts that draw the two maps for *The Machine in the Wood* in the
black-ink-on-paper style of an early-'80s B/X module. They render at 3x and
downsample, so the line art is clean enough to drop straight into the PDF.

## Files

| File | What it does |
|---|---|
| `osrmaps.py` | Shared drawing kit: fonts, paper, frame, cartouche, compass, scale bar, grid, stairs, doors, water hatch, keyed markers. Everything works in design pixels; it handles the supersampling. |
| `gen_hush.py` | The Hush wilderness, a pointy-top hex map (X1 style). Forest, mountains, the Blight ring, Quelling, the Wreath, keyed sites H-1..H-6, the salvage road. → `hush_map.png` |
| `gen_wreath.py` | The Wreath dungeon (B-series grid style): the ring upper level (iris entry, gallery, the Rejected cells, the wet crawl, the stairs, the white-tree courtyard) and the lower precursor level (Cold Gallery, Drone Bay, Governor, Coolant Sump, Operator's Hall). → `wreath_map.png` |
| `gen_all.py` | Runs both. |

> **Hex terrain icons note:** the wilderness *terrain* tiles in `icons/hex/` are
> now the external **Hexographer Black & White Classic** set (see
> `../CREDITS.md`), not generated art. `gen_hexicons.py` still generates the
> *feature/settlement* glyphs (Castle, Town, Ruins, …) but its old terrain
> output (Forest, Mountains, Grasslands, Pines, …) is superseded — **do not
> blindly re-run it**, as it would overwrite the Hexographer terrain tiles and
> re-introduce removed names. Re-run only after pruning its terrain outputs.

## Requirements

Python 3 and Pillow.

```
pip install pillow          # if not already present
```

Optional, for nicer labels: the **EB Garamond** and **Jost** fonts (the same
faces the module PDF uses). The scripts fall back to DejaVu if those are missing.

## Run

```
cd maps
python3 gen_all.py
```

This (re)writes `hush_map.png` and `wreath_map.png` in this folder.

## Tweaking

Both scripts are data-first; you change the map by editing a few tables near the
top, not the drawing code.

- **Move or rename a Hush site:** edit the `SITES` dict in `gen_hush.py` (keys are
  `(column, row)` on the hex grid). `MOUNTAIN`, `ROAD`, and `PATHS` control terrain
  and trails. The Blight ring is computed automatically from the Wreath's hex.
- **Re-key the dungeon:** edit the `rooms` and `corrs` lists in `gen_wreath.py`
  (`lower()`), or the radii and feature angles in `upper()`.
- **Resize for print:** change `WPX, HPX` at the top of each script. Raise `SS` in
  `osrmaps.py` for higher resolution (slower, more memory).
- **Palette:** set `PALETTE` at the top of `osrmaps.py`. Ships as `"bw"` (black ink
  on white, the X1-interior look, photocopy-friendly). `"cream"` gives warm paper;
  `"blue"` gives the B2 blue-graph look. Re-run `gen_all.py` after changing it.
- **Fonts:** edit `_FONTS` in `osrmaps.py` to point at whatever you have installed.

## Icons & the Map Maker

The interactive editor `../OSR Map Maker.html` stamps icons from two sets:

- **Dungeon symbols** — the 86 classic high-res PNGs, copied into `icons/dungeon/`.
- **Hex terrain** — 23 icons (Castle, Forest, Mountains, Swamp, Volcano, ...) in the
  old Expert-set style, generated by `gen_hexicons.py` into `icons/hex/`. Run it to
  regenerate or edit; it also writes `icons/_hexicons_sheet.png` (a contact sheet).

`build_icons_js.py` base64-encodes both folders into `../icons.js`, which the Map
Maker loads. Using data URIs (not file paths) keeps the canvas un-tainted so
**Export PNG** works. Re-run it after adding or changing any icon:

```
cd maps
python3 gen_hexicons.py     # (re)build hex terrain icons
python3 build_icons_js.py    # bundle icons/ -> ../icons.js
```

To add your own icon: drop a transparent-background PNG into `icons/dungeon/` (or
`icons/hex/`), re-run `build_icons_js.py`, reload the tool. Multi-tile dungeon
icons read their size from the filename (e.g. `...2x2.png`).

The **Python** dungeon generator uses the same icons: `gen_wreath.py` calls
`osrmaps.stamp('dungeon', 'Door1x1', cx, cy, box)` to paste a PNG onto the map
(via alpha compositing), so the printed module maps and the editor share one
symbol set. `osrmaps.load_icon` caches them; only transparent-background PNGs
composite cleanly.

## Style notes

The look copies the conventions of classic old-school book design (see the study
in the `Layout` folder): thick black walls, a 10-ft grid, keyed discs, a titled
cartouche, a segmented scale bar, a compass rose, and a boxed legend.
